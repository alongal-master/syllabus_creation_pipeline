Here: FE101 — Introduction to Modern JavaScript & TypeScript, a bridge course for Python programmers entering frontend and full-stack engineering. AI-tutor delivered, 1-on-1, self-paced.

🎯 Six Course Mastery Outcomes (graduate skills)
#Outcome
LO1Transition algorithmic problem-solving skills from Python to JavaScript: given a short algorithmic problem already solved in Python (e.g., filter a list, count items by condition), rewrite it in JavaScript without syntax errors and identify two key differences in how JS and Python handle the same logic.
LO2Understand and apply core JavaScript syntax and language fundamentals: write a 20–40 line JS program from scratch using variables (let/const), conditionals, loops, and functions — with no Python syntax present — and correctly predict the output of a program that uses type coercion or loose equality.
LO3Work confidently with common data structures: transform an array of objects using map, filter, and reduce; add, access, update, and delete object properties; and destructure both arrays and objects in function parameters.
LO4Write short reusable functions including function expressions, arrow functions, and anonymous callbacks: rewrite a named function declaration as a function expression and as an arrow function; pass a callback to a higher-order function like forEach or setTimeout; and distinguish passing a function reference from calling a function.
LO5Understand the fundamentals of synchronous and asynchronous programming in JavaScript: correctly predict the console output order of a program that mixes synchronous code with a setTimeout callback; write a function that uses async/await to handle a Promise; and apply try/catch for async error handling.
LO6Read and interpret basic TypeScript type annotations and error messages on variables, arrays, and functions: given a TypeScript file with a type error, identify which line is wrong, explain why TypeScript is rejecting it, and fix it — for scenarios involving primitive types, typed arrays, and function parameter/return types.
📄 Course context & structural rules

Target learner: adult Python programmer (bootcamp or self-taught) pivoting to web or AI engineering. No prior JS or TypeScript experience. Structural rules: every Unit 1–2 lesson must anchor JS syntax to Python equivalent; no lesson may introduce more than two new concepts simultaneously; async scaffolded over at least three sequential lessons (call stack → event loop → callbacks → Promises → async/await); TypeScript introduced only after all JS patterns are in place; lesson titles 2–6 words, sentence case, no "Introduction to" patterns; course must include a motivation statement for career-switching adults.

🧭 Agent: Learning Arc Designer
i

What it is: The learning arc is the course's identity and learning flow — what changes from lesson to lesson and unit to unit, why it changes, and how the course systematically builds toward mastery.

Learning arc: The Language Translator

Every lesson in Units 1 and 2 anchors new JavaScript syntax and behavior directly to its Python equivalent — naming what maps across, what changes shape, and what is genuinely new — so students spend zero time feeling like beginners and maximum time building fluency. From that foundation, Unit 3 introduces JavaScript's most distinctive concept: single-threaded asynchronous execution, which Python largely conceals and which must be scaffolded step by step. Unit 4 completes the arc by adding TypeScript's type layer on top of JS patterns students have already mastered, connecting back to the type coercion surprises of Unit 1 and showing why static types matter.

  • JS foundations: Rapid Python-to-JS orientation using Python as a scaffold — every new concept explicitly anchored to its Python equivalent or flagged as genuinely new.
  • Arrays, objects, and functions: JS data layer deepened with destructuring, spread/rest, and higher-order array methods, grounded in Python parallels. Functions graduate to first-class values, callbacks, and closures.
  • Async JavaScript: The course's hardest conceptual shift — from synchronous top-to-bottom execution to JS's single-threaded non-blocking model. Scaffolded: call stack → event loop → callback hell → Promises → async/await → error handling → parallelism.
  • TypeScript types: TypeScript's type annotation layer added on top of mastered JS patterns. Every concept connects back to a specific JS behavior — especially type coercion — to show why static types exist.

Organizing principle: Use Python as a scaffold — explicitly name what translates, what changes shape, and what is genuinely new — so learners spend zero time feeling like beginners and maximum time building real fluency.

📚 8 Core Concepts — Descriptions
IDConceptDescription / role in course
cc1 ⚠️JS type system & coercionJavaScript performs implicit type coercion when operators are applied to mismatched types — producing results like '3'+3='33' that Python would never allow silently. Primary "this language works differently" moment; underpins equality behavior, common bugs, and the entire rationale for TypeScript. Threshold.
cc2Equality: == vs ===JS's == coerces types before comparing; === compares value and type without coercion. Practical rule: always use ===. Direct, testable consequence of cc1.
cc3 ⚠️Scope and closuresScope determines which variables are accessible at any point. A closure retains access to variables from its enclosing scope after that scope has returned. Prerequisite to reasoning about callbacks and async state. Threshold.
cc4 ⚠️Functions as first-class valuesIn JS, functions are values — assignable to variables, passable as arguments, returnable from other functions. This mental model shift enables callbacks, higher-order functions, and the entire async pattern. Threshold.
cc5Callbacks and higher-order functionsA callback is a function passed as an argument to be called later. A higher-order function takes a function as input or returns one. Powers map/filter/reduce and async result delivery.
cc6 ⚠️The event loop and async executionJavaScript runs on a single thread. The event loop enables non-blocking execution by deferring callbacks to a queue that runs after the call stack empties. No direct Python equivalent — requires rebuilding the mental model of how code runs. Threshold.
cc7Promises and async/awaitA Promise represents a value available in the future (or an error). async/await is syntactic sugar over Promises letting async code read like sequential code. Both built on the event loop.
cc8TypeScript static types and annotationsTypeScript adds a compile-time type layer to JavaScript. Type annotations declare what type a variable/parameter/return value must hold; the compiler flags violations before code runs. Direct solution to JS type coercion bugs.

Course mastery outcomes (graduate skills)

These are the human-defined outcomes the pipeline was given; the syllabus must cover all six.

#Outcome
LO1Transition algorithmic problem-solving skills from Python to JavaScript: given a short algorithmic problem already solved in Python (e.g., filter a list, count items by condition), rewrite it in JavaScript without syntax errors and identify two key differences in how JS and Python handle the same logic.
LO2Understand and apply core JavaScript syntax and language fundamentals: write a 20–40 line JS program from scratch using variables (let/const), conditionals, loops, and functions — with no Python syntax present — and correctly predict the output of a program that uses type coercion or loose equality.
LO3Work confidently with common data structures: transform an array of objects using map, filter, and reduce; add, access, update, and delete object properties; and destructure both arrays and objects in function parameters.
LO4Write short reusable functions including function expressions, arrow functions, and anonymous callbacks: rewrite a named function declaration as a function expression and as an arrow function; pass a callback to a higher-order function like forEach or setTimeout; and distinguish passing a function reference from calling a function.
LO5Understand the fundamentals of synchronous and asynchronous programming in JavaScript: correctly predict the console output order of a program that mixes synchronous code with a setTimeout callback; write a function that uses async/await to handle a Promise; and apply try/catch for async error handling.
LO6Read and interpret basic TypeScript type annotations and error messages on variables, arrays, and functions: given a TypeScript file with a type error, identify which line is wrong, explain why TypeScript is rejecting it, and fix it — for scenarios involving primitive types, typed arrays, and function parameter/return types.

Chosen learning arc

The Language Translator — Every lesson in Units 1 and 2 anchors new JavaScript syntax and behavior directly to its Python equivalent — naming what maps across, what changes shape, and what is genuinely new — so students spend zero time feeling like beginners and maximum time building fluency. From that foundation, Unit 3 introduces JavaScript's most distinctive concept: single-threaded asynchronous execution, which Python largely conceals and which must be scaffolded step by step. Unit 4 completes the arc by adding TypeScript's type layer on top of JS patterns students have already mastered, connecting back to the type coercion surprises of Unit 1 and showing why static types matter.

Unit 1: JS foundations (9 lessons)

A rapid orientation to JavaScript using Python as a scaffold. Students who already think like programmers discover what maps directly from Python, what changes shape, and what is genuinely new — building fluency and confidence before any advanced patterns are introduced.

IDTitleMastery outcomes
u1l1JS for Python developers
  • Identify at least three categories of software engineering work where JavaScript is the required language and Python alone is insufficient.
  • Distinguish the role of FE101 in the overall learning path from Python programmer to web/AI software engineer.
  • Predict what a developer can build differently after completing this course versus before it.
u1l2Variables and constants
  • Write variable declarations using let and const and predict which will throw an error when reassigned.
  • Distinguish block scope (let/const) from function scope (var) and identify at least one bug var can cause that let prevents.
  • Compare JS variable declaration syntax to Python's assignment syntax and name the key structural difference.
u1l3Types and type coercion
  • Identify the six JS primitive types and distinguish them from Python equivalents.
  • Predict the output of coercion expressions such as '5' + 3, '5' - 3, 0 == false, and !!'' without running the code.
  • Apply typeof to inspect a value's type and interpret the result, including the typeof null quirk.
u1l4Equality and comparison operators
  • Distinguish == (loose equality with coercion) from === (strict equality without coercion) and apply the correct operator in a given context.
  • Predict the boolean result of expressions using ==, ===, >, <, >=, and <= including cases where types differ.
  • Identify which JS values are falsy (0, '', null, undefined, NaN, false) and apply this to reading conditional expressions.
u1l5Control flow in JS
  • Write if/else, for, while, and for...of loops in JS with correct syntax (curly braces, semicolons) without reverting to Python patterns.
  • Compare JS for...of to Python's for item in list and identify what is structurally equivalent and what differs.
  • Fix syntax errors in JS control flow code caused by missing braces, incorrect loop headers, or Python-style syntax.
u1l6Functions: declarations and expressions
  • Write a function using both declaration syntax and expression syntax and compare each to Python's def syntax.
  • Distinguish hoisting behavior: identify that function declarations are hoisted but function expressions are not.
  • Assign a named function to a variable and pass it as an argument to another function without calling it (no parentheses). [cc4 — first assessable outcome]
u1l7Strings and template literals
  • Apply common JS string methods (toUpperCase, toLowerCase, includes, slice, split, trim) to transform and inspect strings.
  • Write template literals using backtick syntax and ${ } interpolation and compare them to Python f-strings.
  • Fix string concatenation bugs by replacing + concatenation with template literals where appropriate.
u1l8Scope and variable lifecycle
  • Predict whether a variable declared with let, const, or var is accessible at a given point in a program by tracing its scope.
  • Identify the difference between Python's function-level scope and JS's block-level scope and describe a case where this difference matters.
  • Distinguish a ReferenceError caused by accessing a variable outside its scope from a TypeError caused by incorrect usage.
u1l9JS syntax in practice
  • Write a complete JS program (20–40 lines) that uses variables, control flow, at least one function, and string manipulation to solve a given problem — without any Python syntax.
  • Identify and fix at least three distinct JS syntax or type-coercion bugs in a provided program.
  • Apply === instead of == throughout and use const/let appropriately, with no var usage.
Unit 2: Arrays, objects, and functions (10 lessons)

The JS data layer: arrays, objects, and the function patterns that operate on them. Students learn destructuring, spread/rest, and higher-order array methods — grounded in Python list and dict parallels — and graduate from basic functions to first-class function values, callbacks, and closures.

IDTitleMastery outcomes
u2l1Arrays and array methods
  • Create and mutate JS arrays using push, pop, shift, unshift, splice, and slice and predict the return value of each operation.
  • Compare JS array mutation behavior to Python list methods and identify one key difference in how slice works across the two languages.
  • Distinguish methods that mutate the original array from those that return a new array and apply the correct method for a given task.
u2l2Objects and properties
  • Create JS objects using literal syntax and access properties using both dot notation and bracket notation.
  • Add, update, and delete properties on an existing object and predict the state of the object after each operation.
  • Compare JS objects to Python dictionaries: identify what is equivalent and what differs (dot notation, method syntax).
u2l3Array and object destructuring
  • Apply array destructuring to extract values from an array into named variables in a single statement.
  • Apply object destructuring to extract specific properties from an object, including with renamed variables.
  • Write a function that uses object destructuring in its parameter list to receive a configuration object cleanly.
u2l4Spread and rest operators
  • Use the spread operator (...) to copy arrays, merge arrays, and copy or merge objects without mutating the originals.
  • Write a function that uses rest parameters (...args) to accept a variable number of arguments and compare this to Python's *args.
  • Distinguish spread (expanding an iterable) from rest (collecting arguments) by identifying which role ... plays in a given code example.
u2l5Functions as values
  • Assign a function to a variable and pass that variable as an argument to another function without calling it (no parentheses), and contrast this with the bug that occurs when parentheses are added.
  • Distinguish a function reference (myFn) from a function call (myFn()) and identify which is required in a callback context.
  • Compare JS first-class functions to Python: note that Python functions are also first-class values, but that idiomatic JS relies on this pattern far more heavily throughout the language.
u2l6Closures
  • Write a factory function that returns another function, and call the returned function to demonstrate that it retains access to variables from its enclosing scope.
  • Identify which outer-scope variables a closure captures and predict their values when the closure is called, including cases where the outer scope has already returned.
  • Distinguish the mental model 'functions get a copy of outer variables at definition time' (incorrect) from 'functions retain a live reference to outer variables' (correct) by tracing a closure example step by step.
u2l7Arrow functions
  • Rewrite any function expression as an equivalent arrow function, including the implicit return shorthand for single-expression bodies.
  • Identify when implicit return applies and when curly braces and an explicit return statement are required.
  • Apply the correct arrow function syntax as an inline callback argument to a higher-order function.
u2l8map, filter, and reduce
  • Apply map, filter, and reduce to transform arrays using arrow function callbacks and predict the return value of each.
  • Chain map and filter together to solve a two-step array transformation problem.
  • Compare JS map/filter/reduce to Python's list comprehensions and map()/filter() and identify one structural advantage of each approach.
u2l9Writing higher-order functions
  • Write a function that accepts a callback parameter and calls it internally, using correct syntax (no extra parentheses on the parameter).
  • Apply forEach for side-effect iteration and distinguish it from map (which returns a new array).
  • Write a custom sort comparator as a callback to Array.prototype.sort and predict the sort order from the comparator's return value.
u2l10Data pipeline in JS
  • Solve a multi-step data transformation problem using a combination of map, filter, reduce, and destructuring on an array of objects.
  • Write a higher-order utility function (e.g., a custom groupBy or countBy) that accepts an array and a callback and returns a transformed result.
  • Fix a buggy program that misuses mutation, incorrect callback invocation, or missing return values in reduce.
Unit 3: Async JavaScript (9 lessons)

The hardest conceptual shift in the course: from synchronous top-to-bottom execution to JavaScript's single-threaded, non-blocking execution model. Each lesson builds one layer of the mental model — call stack, event loop, callbacks, Promises, async/await — before the next is introduced.

IDTitleMastery outcomes
u3l1The call stack
  • Trace the execution order of a synchronous JS program by simulating the call stack state at each function call and return.
  • Predict the exact console output order of a synchronous program that uses nested function calls.
  • Identify what 'blocking' means in a synchronous context and why a long-running synchronous operation would freeze a web application.
u3l2The event loop
  • Predict the console output order of a program that mixes synchronous code with setTimeout callbacks, including setTimeout with delay 0.
  • Describe the event loop in plain terms: call stack empties, callback queue checked, callback moves to stack.
  • Distinguish 'non-blocking' from 'parallel': identify that JS does not run two things simultaneously but defers work to be done later.
u3l3Callbacks and async timing
  • Write an async operation using setTimeout that uses a callback to deliver its result, following the Node.js-style (error, result) callback convention.
  • Predict the execution order and output of nested setTimeout callbacks (callback inside a callback).
  • Identify why deeply nested async callbacks ('callback hell') become unmaintainable and describe what makes the pattern problematic.
  • Identify which outer-scope variable an async callback closes over and predict its value when the callback fires. [cc3 reinforcement]
u3l4Promises: resolve and reject
  • Create a Promise using new Promise((resolve, reject) => {...}) that resolves or rejects based on a condition.
  • Consume a Promise using .then for the resolved value and .catch for the rejected value.
  • Distinguish the three states of a Promise (pending, fulfilled, rejected) and predict which handler runs given a specific resolution.
u3l5Chaining promises
  • Write a Promise chain using multiple .then calls where each handler receives the return value of the previous one.
  • Apply .finally to run cleanup code regardless of whether a Promise resolved or rejected.
  • Rewrite nested Promise callbacks (a Promise inside a .then) as a flat Promise chain.
u3l6async and await
  • Write an async function using the async keyword and use await to pause execution until a Promise resolves.
  • Rewrite a .then/.catch Promise chain as an equivalent async/await function.
  • Predict what an async function returns (always a Promise) and identify what value a caller receives when it awaits an async function.
u3l7try/catch and async errors
  • Wrap an await expression in try/catch to handle a rejected Promise and distinguish this from unhandled rejection.
  • Apply .catch at the end of a Promise chain and predict which rejections it will catch.
  • Identify what happens when an async function throws an error without a try/catch — and how the caller receives the error.
u3l8Promise.all and parallel async
  • Write code using Promise.all to run multiple Promises in parallel and collect all results in a single await expression.
  • Distinguish Promise.all (fails fast on first rejection) from Promise.allSettled (waits for all, reports each outcome).
  • Predict the output and timing of a program that uses Promise.all vs sequential await calls for the same operations.
u3l9Async execution: full picture
  • Predict the exact console output order of a program that mixes synchronous code, setTimeout, and async/await.
  • Rewrite a callback-based async sequence as a clean async/await function with full error handling using try/catch.
  • Fix an async function that has at least two bugs: a missing await, and an unhandled rejection.
Unit 4: TypeScript types (5 lessons)

A focused introduction to TypeScript's type annotation layer, built entirely on JS patterns students have already mastered. Every TypeScript concept is connected back to a specific JS behavior — especially type coercion — to show why static types exist and what problems they solve.

IDTitleMastery outcomes
u4l1TypeScript as a JS superset
  • Distinguish TypeScript from JavaScript: identify that TypeScript compiles to JS and that type annotations do not exist at runtime.
  • Identify a JS coercion bug (from Unit 1) that TypeScript would have caught at compile time, and explain why.
  • Predict which TypeScript annotations are correct and which will produce a compile error for a given set of variable declarations.
u4l2Types in function signatures
  • Add type annotations to function parameters and return types for functions that accept and return primitives.
  • Identify what TypeScript error is produced when a function is called with an argument of the wrong type, and fix it.
  • Apply the void return type to functions that produce side effects and have no return value.
u4l3Typed arrays and union types
  • Declare typed arrays using the T[] syntax and predict which array operations TypeScript will accept or reject based on the array's element type.
  • Write a union type annotation (string | number) for a variable that can hold more than one type and identify when this is appropriate.
  • Use typeof type narrowing to safely work with a union type variable inside a conditional block. [added in v3 per rb12]
u4l4Reading TypeScript errors
  • Read a TypeScript error message and identify the specific line, expected type, and received type from the error text.
  • Fix at least three categories of TypeScript errors: wrong argument type, missing return type, and assigning the wrong type to a typed variable.
  • Distinguish a TypeScript compile-time error from a JavaScript runtime error and explain why catching errors at compile time is preferable.
u4l5TypeScript in practice
  • Identify all TypeScript errors in a 30–50 line TypeScript program that contains primitive type mismatches, function signature errors, typed array violations, and union type misuse.
  • Fix every error in the file and produce a version that compiles without TypeScript errors, including applying typeof type narrowing where the union type misuse requires it.
  • Complete a TypeScript program spanning all four error categories — primitive type mismatches, function signature errors, typed array violations, and union type misuse — producing a compiling file, and name one skill gained in this course that you could not do when you started.
