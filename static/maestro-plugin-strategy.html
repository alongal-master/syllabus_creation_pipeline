Maestro-controlled · the contract

Every plugin speaks the same language.

A shared interface is what makes it compose. Any new plugin built to this contract just slots in — the tutor already knows how to talk to it.

  • manifestIdentity. Declares what the plugin is, the inputs it accepts, what it submits back.
  • mountRender. The platform hands over a container; the plugin renders into it.
  • inputsConfigure. The lesson passes in plugin-specific data — cells for a spreadsheet, nodes for a diagram.
  • submit()Return. The plugin returns the artifact the tutor needs. The button lives in Maestro's UI, not the plugin.
  • unmountClean shutdown.

Real world · the challenge

It's messy by nature — that's the cost.

Connectors buy us reach into real tools, but each one is bespoke. That's the trade-off behind the build-real-stuff side.

  • →Artifacts are wildly different: a video, a website, a model, an ad campaign, a spreadsheet, a diagram.
  • →Environments differ: VS Code, a browser, Google Ads, QuickBooks, real cloud platforms.
  • →The tutor's visibility is limited — work happens in tools Maestro doesn't own.
  • →Each course needs its own connector to the learner's real work.
So we invest deliberately. A module is built only where a course genuinely needs to touch the real world — everywhere else, plugins do the job.

the roadmap

Top priorities to develop.

Where we invest first — the foundations everything else builds on.

Priority 01

Plugins infrastructure

The reusable, agnostic building blocks. Define the plugin contract and ship the first set — spreadsheet, notebook, SQL workspace, diagram — so any lesson can compose them.

Priority 02

Project-based learning pedagogy

How build-real-stuff courses actually teach: scoping, milestones, and tutor review of real artifacts. The method behind the projects, not just the tooling.

Priority 03

Connectors — start with GitHub

The links into real tools. Begin with a GitHub connector so the tutor can read the learner's repo — then expand to the next environment from there.
